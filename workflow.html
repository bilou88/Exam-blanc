Workflow Git

Voici un guide court et clair, dans le même style que ta page "Introduction à Git" : phrases courtes, structure simple et exemples de commandes.

1. Créer une branche de fonctionnalité

Partir de la branche principale (par ex. main) puis créer une branche dédiée : 

git checkout -b feature/ma-fonction

Commiter souvent avec des messages lisibles :

git add .
git commit -m "description claire de ce qui a été fait"

2. Pousser la branche

Envoyer la branche vers le dépôt distant pour partager le travail ou lancer l'intégration continue : 

git push -u origin feature/ma-fonction

3. Ouvrir une Pull Request (PR)

Sur GitHub, ouvre une PR depuis ta branche vers main. Explique l'objectif, liste les changements et indique comment tester.

4. Revue et corrections

Les relecteurs laissent des commentaires. Fais les corrections sur ta branche, puis repousse : la PR se mettra à jour automatiquement.

5. Merge et nettoyage

Une fois approuvée et passée l'intégration, merge dans main. Supprime ensuite la branche distante. 

git checkout main
git pull origin main
git merge --no-ff feature/ma-fonction
git push origin main
# supprimer la branche distante :
git push origin --delete feature/ma-fonction

Résumé : créer une branche, travailler, pousser, ouvrir une PR, faire la revue, puis merge.
